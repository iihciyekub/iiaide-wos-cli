iiaide-wos Command Cheatsheet

Default mode is one managed project per working directory. State is stored in ./.iiaide-wos-cli/.

Project

Initialize / inspect

iiaide-wos init
iiaide-wos workspace
iiaide-wos list
iiaide-wos tasks
iiaide-wos latest
iiaide-wos show
iiaide-wos path
iiaide-wos validate
iiaide-wos clear
Auth

SID and browser

iiaide-wos check
iiaide-wos sid --from-browser --debug
iiaide-wos sid-pool
iiaide-wos auth login --provider must
iiaide-wos auth monitor --provider must
Query

UUID discovery

iiaide-wos query build --expr 'PY=(2026)'
iiaide-wos query build --expr 'PY=(2026)' --force
iiaide-wos query batch --expr 'PY=(2025)' --expr 'PY=(2026)'
cat > queries.txt <<'EOF'
# Two-dimensional materials project: compare recent yearly result sets
TS=("two-dimensional materials") AND PY=(2025)
TS=("two-dimensional materials") AND PY=(2026)

# Atomic thickness theory slice
TS=(("two-dimensional materials" OR graphene OR "transition metal dichalcogenide*") AND ("atomic thickness" OR monolayer) AND (theor* OR model* OR simulation*))
EOF
iiaide-wos query batch --expr-file ./queries.txt
iiaide-wos query parse --text "2026 AI safety papers"
iiaide-wos query ids --wosid "WOS:000000000000001"
iiaide-wos query ids --csv "./input/ids.csv"
iiaide-wos query batch --expr-file "./queries.txt" --jsonl

query build prints compact single-line JSON with uuid, url, count, queryText, and cached. The same successful task/query text is reused from SQLite by default; use --force to query WOS again. query batch prints JSONL by default, accepts repeated --expr, and runs uncached expressions sequentially in one WOS browser session. A queries.txt file is plain UTF-8 text: one WOS advanced-search expression per line; blank lines and lines starting with # are ignored.

Record

Relation UUIDs

iiaide-wos record relations --wosid "WOS:000000000000001" --type citations
iiaide-wos record collect --wosid "WOS:000000000000001" --json
iiaide-wos record shared --wosid "WOS:000000000000001" --with "WOS:000000000000002"
Ingest

SQLite

iiaide-wos query ingest --expr 'PY=(2026)' --description "2026 search"
iiaide-wos record ingest --wosid "WOS:000000000000001" --type references
iiaide-wos db uuid --uuid "<uuid>" --json
iiaide-wos db wosid --wosid "WOS:000000000000001" --json
iiaide-wos db list --uuid "<uuid>" --context --json
iiaide-wos db list --wosid "WOS:000000000000001" --type references --json
iiaide-wos db list --wosid "WOS:000000000000001" --type references --context --json
iiaide-wos record ingest --wosid "WOS:000000000000001" --type citations --force --json
iiaide-wos db context --wosid "WOS:000000000000001" --type references --json
iiaide-wos db audit-html
iiaide-wos db audit-export

Relation ingest collects WOSIDs from at most the first 6 relevance pages, queries those WOSIDs, and marks the relation UUID as exportMode=front-scroll-wosid. Confirmed zero-count relations are stored and reused. Use db list for the ordered stored WOSID list, with optional record context.

Export

TXT / BibTeX

iiaide-wos run --uuid "<uuid>"
iiaide-wos run --uuid "<relation-uuid>" --ref-query
iiaide-wos bib --uuid "<uuid>"
iiaide-wos batch-run --search-root "."
Import

CSV

iiaide-wos import --csv "./input/wosids.csv"
Legacy

Optional multi-task mode

iiaide-wos run --uuid "<uuid>" --tasks-root "/path/to/tasks" --task "txt-export"
iiaide-wos query ingest --expr 'PY=(2026)' --tasks-root "/path/to/tasks" --task "query-db"
